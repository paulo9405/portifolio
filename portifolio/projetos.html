Projeto Estacionamento

Projeto feito com base no conhecimento que adquiri ate o momento

Um sistema de estacionamento feito em Django que tem a capacidade de gerenciar veículos, pessoas, pagamentos diários e pagamentos mensais, gerando relatórios em PDF ou CSV.

O maior desafio encontrado durante o desenvolvimento desse projeto foi a questão evolucionária do banco de dados, tentando fazer com que o mesmo acompanhe as evoluções do sistema e tenha um histórico de alterações de banco de dados que possa ser versionado via source control (git). O Django oferece a ferramenta "django migrations" que pode não ser muito intuitiva quando se tem o contato pela primeira vez, mas assim que você entende o seu funcionamento, tudo roda como esperado.

Sistema de cadastro e escalas de trabalho e folga de médicos com Python | Django | Bootstrap (com algumas validações).

A - Só poderá preencher a escala se não houver folga para o médico naquele dia da semana.

Projeto API com Django Rest Framework

Criei uma API REST para cadastro de ativos e Ações usando Python | Django | Django Rest Framework, Com algumas validações bem interessantes.

1. Cadastrar ativos Os ativos devem conter no mínimo as informações abaixo:
Nome - uma denominação para este ativo.
Modalidade - renda fixa, renda variável ou cripto.

2. Fazer aplicações e resgates em um ativo:
Aplicação é quando o usuário aporta dinheiro em um ativo.
Resgate é quando o usuário retira dinheiro de um ativo.
Aplicações e resgates são transacionais e imutáveis. Uma vez realizada não há como alterar.
As aplicações e resgates devem conter no mínimo as informações abaixo:
Ativo - O ativo ao qual a aplicação/resgate se refere.
Data de solicitação - a data em que a aplicação/resgate foi solicitada.
Quantidade - número de ativos que foram aplicados/resgatados.
Preço unitário - preço unitário do ativo na aplicação/resgate.
Usuários podem fazer aplicações/resgate em ativos cadastrados por qualquer usuário.
Exemplo:
O usuário A cadastra um ativo chamado BITCOIN e faz uma aplicação de mil reais.
O usuário B pode aplicar no ativo BITCOIN também, pois ele já foi cadastrado pelo usuário A.

3. Visualizar o saldo da sua carteira de investimentos
Você pode decidir onde e como mostrar a informação.
O saldo da carteira é o somatório de saldos investidos em cada um dos ativos.
Usuários não podem:
Ver aplicações e resgates de outros usuários.
Ver o saldo da carteira de outros usuários.
Outros detalhes da API:
Testes
As funcionalidade principais contem testes escritos.

4. Extras: Permitir os usuários listar ativos por tipo (renda fixa, renda váriavel, cripto).
Salvar o endereço de IP do usuário que fez uma aplicação/resgate.
Sistema de adição de taxa de custódia, administração, tarifa de saque.
